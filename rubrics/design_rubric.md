# Design and Originality Rubric

Use this when the risk is AI slop, generic UI, purple-gradient sameness, or weak product taste.

## Design Quality Checks

Score 0-5.

### Coherence

- Do screens feel related?
- Are visual rules repeated intentionally?
- Does the hierarchy guide the eye?

### Specificity

- Does the interface fit this product's domain?
- Are labels and empty states specific?
- Are visual metaphors appropriate?

### Taste

- Are colors restrained and intentional?
- Are gradients, shadows, and glass effects justified?
- Is there a clear reason for every decorative element?

## Originality Checks

Score 0-5.

Fail originality if:

- UI looks like a default Tailwind dashboard.
- Uses generic purple/blue gradients without purpose.
- Uses stock SaaS cards with interchangeable labels.
- Contains generic marketing copy.
- Has no product-specific interaction model.

## Anti-Slop Rules

The Evaluator should ask:

1. Could this UI belong to any random startup?
2. Are there custom decisions a human designer would recognize?
3. Is the interface solving the product problem or decorating around it?
4. Are there signs the Generator accepted defaults too quickly?

If answers indicate generic output, fail Design and Originality even if functionality works.
